Concept C# : static readonly Color

static readonly declare des constantes initialisees une seule fois au demarrage. Comme des pots de peinture etiquetes dans un atelier : on les prepare une fois, puis on y pioche. PAColor est passe de bleu clair (0.3, 0.5, 1) a bleu fonce (0.15, 0.25, 0.7) pour le differencier visuellement de ShieldColor (0.3, 0.6, 1) qui reste bleu clair.

Logique : Systeme d'empilement

StackOffset definit l'espacement vertical entre textes empiles (0.5 unite). _activeLabels est un dictionnaire static qui mappe chaque Label3D actif vers sa position d'origine. Comme une liste de post-its colles sur un tableau : on sait ou chacun a ete pose pour empiler les suivants au-dessus.

Logique : Nettoyage des labels invalides

Avant chaque spawn, on parcourt _activeLabels pour retirer les labels detruits (par QueueFree). IsInstanceValid verifie qu'un objet Godot n'a pas ete libere. On ne peut pas modifier un dictionnaire pendant son iteration, d'ou la liste toRemove intermediaire.

Logique : Comptage des textes a la meme position

On compare les positions X et Z (pas Y, car on empile verticalement) avec une tolerance de 0.1f. Si 2 textes sont deja actifs sur ce pion, le 3e apparaitra 1.0 unite plus haut (2 * 0.5). C'est comme empiler des assiettes : chaque nouvelle assiette se pose sur la pile existante.

Godot : Label3D Billboard

Billboard = Enabled fait que le texte fait toujours face a la camera, quelle que soit l'orientation. NoDepthTest = true le rend visible meme a travers les objets 3D. Ideal pour des indicateurs de gameplay qui doivent etre toujours lisibles.

Godot : Tween animations

CreateTween() cree une animation procedurale. SetParallel(true) fait jouer les tweens suivants en meme temps (montee + fondu). SetParallel(false) repasse en sequentiel pour que le callback de destruction s'execute apres. Le callback utilise une lambda C# qui nettoie le dictionnaire avant de detruire le label.

Pattern : Event handler pour interception de degats

AbsorbDamage est branche sur l'event OnBeforeTakeDamage du pawn. C'est le pattern Observer : quand des degats arrivent, tous les effets abonnes peuvent les modifier AVANT qu'ils ne soient appliques. Le DamagePacket est un objet mutable passe par reference — chaque effet peut reduire FinalAmount.

Logique : Calcul d'absorption

Math.Min(ShieldAmount, packet.FinalAmount) prend le minimum entre le shield restant et les degats entrants. Si 5 degats arrivent et le shield a 3 points : absorbe 3, il reste 2 degats a subir en PV. Si le shield tombe a 0, DurationRemaining = 0 marque l'effet pour suppression.

Logique : Nouveau floating text shield (NOUVEAU)

La ligne ajoutee fait apparaitre un texte bleu clair (ShieldColor) au-dessus du pion montrant combien le bouclier a absorbe. Grace au systeme d'empilement de FloatingText, ce "-3" bleu clair ne se superposera pas au "-2" rouge (degats HP) qui apparait juste apres.

Concept Shader : uniform vs instance uniform

uniform est partage par TOUTES les instances utilisant ce materiau (ex: la couleur de fond est la meme pour toutes les barres). instance uniform est propre a CHAQUE instance (chaque pion a son propre health et shield). C'est comme un moule a gateau (uniform) vs la garniture de chaque gateau (instance uniform).

Godot Shader : Billboard dans vertex()

Le calcul de MODELVIEW_MATRIX fait pivoter la barre pour toujours faire face a la camera. On remplace l'orientation du modele par celle de la camera inverse (INV_VIEW_MATRIX) tout en gardant sa position (MODEL_MATRIX[3]).

Logique : Barre a 3 zones (NOUVEAU)

Le fragment shader divise la barre horizontalement via UV.x (0 a gauche, 1 a droite) :
1. UV.x < health → zone HP (couleur rouge/verte)
2. UV.x < health + shield → zone shield (bleu)
3. Le reste → fond (blanc/gris)
Avant la modification, il n'y avait que 2 zones (HP et fond). Le shield se superpose sur la partie HP : health est reduit cote C# pour faire de la place au shield.

Logique : Formule du shield sur la barre de vie

Le shield se superpose sur la zone HP, sans jamais la depasser :
1. healthNorm = HP courants / HP max (ex: 8/10 = 0.8)
2. shieldRatio = shield total / HP courants (ex: 4/8 = 0.5)
3. shieldNorm = min(shieldRatio * healthNorm, healthNorm) = min(0.5 * 0.8, 0.8) = 0.4
4. On envoie au shader : health = 0.8 - 0.4 = 0.4, shield = 0.4
Resultat visuel : [====HP(40%)====][===SHIELD(40%)===][---vide(20%)---]
Le shield mange une partie de la zone HP mais ne depasse jamais au-dela de la zone remplie.

Pattern : Panel UI lecture seule

MonsterOverviewUI herite de MarginContainer et non de Control simple, car il a besoin de marges internes. Il vit EN DEHORS du systeme de layers exclusifs (IBattleUILayer) : il peut etre affiche en meme temps que PlayerTurnUI ou EnemyTurnUI. C'est un overlay independant.

Godot : _Ready() et GetNode<T>

_Ready() est appele quand le noeud entre dans l'arbre de scene. GetNode<T>("chemin") recupere un noeud enfant par son chemin dans la hierarchie. La structure (PanelContainer > MarginContainer_Inner > HBoxContainer_Main) est definie dans le fichier .tscn.

Godot : Creation dynamique de DiamondStatDisplay

Les losanges PA/PM sont crees par code (pas dans le .tscn) et ajoutes au HBoxContainer. AddThemeConstantOverride("separation", 8) definit l'espacement entre les enfants. SizeFlagsVertical = ShrinkCenter centre verticalement le conteneur. Ce pattern est identique a celui de PlayerTurnUI._Ready().

Logique : ShowMonster — pas de toggle

Si on re-clique sur le meme monstre deja affiche, on return immediatement au lieu de fermer le panel. Le toggle (clic pour fermer) est gere par le clic droit dans StatePlayerTurn. On desinscrit l'ancien monstre de OnStatsChanged avant d'inscrire le nouveau pour eviter les fuites d'events.

Godot : Instantiation de boutons d'abilities

Chaque ability du monstre est affichee comme un AbilityButton instancie depuis une PackedScene. La difference avec PlayerTurnUI : les boutons sont Disabled = true (lecture seule, pas cliquables). L'APCostDisplay sous chaque bouton montre le cout en PA.

Pattern : Desinscription d'events

HidePanel desinscrit RefreshStats de OnStatsChanged avant de mettre currentMonster = null. Si on oubliait de desinscrire, le callback continuerait a etre appele apres la fermeture du panel, causant potentiellement des null reference exceptions.

Logique : Mise a jour temps reel des stats

RefreshStats met a jour tous les labels (nom, PV, PA, PM, passif) a partir du pawn courant. Comme elle est branchee sur OnStatsChanged, les stats se mettent a jour en temps reel : si le monstre prend des degats pendant qu'on le regarde, le panel se rafraichit automatiquement.

Glossaire

static class
Classe C# qui ne peut pas etre instanciee. Tous ses membres sont accessibles directement via le nom de la classe (ex: FloatingText.Spawn()).
static readonly
Champ initialise une seule fois au demarrage du programme. Contrairement a const, peut etre initialise avec du code (ex: new Color(...)).
Dictionary<K,V>
Collection de paires cle-valeur. Acces en O(1) par cle. Ici utilise pour mapper des Label3D vers leur position d'origine.
OfType<T>()
Extension LINQ qui filtre une collection pour ne garder que les elements d'un type specifique. Plus propre qu'un if (x is T) dans une boucle.
Tween
Animation procedurale Godot qui interpole une propriete d'un objet entre deux valeurs sur une duree. Alternative au systeme d'AnimationPlayer pour les animations simples.
instance uniform
Variable shader propre a chaque instance de mesh (contrairement a uniform partage par tout le materiau). Set via SetInstanceShaderParameter() en C#.
UV.x
Coordonnee horizontale de texture, de 0 (gauche) a 1 (droite). Utilisee dans le fragment shader pour diviser la barre de vie en zones.
Billboard
Mode d'affichage ou un objet 3D fait toujours face a la camera, quelle que soit l'angle de vue.
QueueFree()
Methode Godot qui marque un noeud pour suppression a la fin de la frame. Plus sur que Free() qui supprime immediatement.
CallDeferred()
Methode Godot qui reporte l'execution d'une methode au prochain cycle idle. Utile pour laisser le layout se recalculer avant d'animer.
DamagePacket
Objet mutable passe a travers les events OnBeforeTakeDamage. Chaque effet (shield, armure, etc.) peut modifier FinalAmount avant l'application des degats.
?. (null-conditional)
Operateur C# qui n'appelle la methode que si l'objet n'est pas null. a?.Method() est equivalent a if (a != null) a.Method().
?? (null-coalescing)
Operateur C# qui retourne la valeur de gauche si non-null, sinon celle de droite. a ?? b = "a, ou b si a est null".
MarginContainer
Conteneur Godot UI qui ajoute automatiquement des marges internes a ses enfants. Utile pour le padding sans CSS.
PackedScene
Reference a une scene Godot (.tscn) pouvant etre instanciee a runtime via Instantiate<T>().
partial class
Permet de diviser une classe C# en plusieurs fichiers. Requis par Godot pour generer du code de binding automatiquement.
IsActionJustPressed
Methode Godot Input qui retourne true uniquement sur la frame ou l'action est declenchee. Attention : retourne true pour TOUS les appelants de la meme frame.
