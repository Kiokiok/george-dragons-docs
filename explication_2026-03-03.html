GameProgress.cs MODIFIE

Systeme central de progression du jeu. Gere un ensemble de flags (drapeaux) qui representent les elements deverrouilles. C'est le "cerveau" qui retient ce que le joueur a accompli. Cette classe a ete entierement reecrite : elle est passee d'une simple coquille vide a un systeme fonctionnel de flags avec evenements.

src/core/GameProgress.cs 
using System;
using System.Collections.Generic;

public static class GameProgress
{
	public const string PATH_LACORSE_NORTH = "path_lacorse_north";

	static readonly HashSet<string> unlockedFlags = new();

	public static event Action<string> OnFlagUnlocked;

	public static void Unlock(string flag)
	{
		if (unlockedFlags.Add(flag))
		{
			OnFlagUnlocked?.Invoke(flag);
		}
	}

	public static bool IsUnlocked(string flag)
	{
		return unlockedFlags.Contains(flag);
	}
}

Annotations

CONCEPT Imports et namespaces (L1-2)

Les directives using sont comme des laissez-passer pour acceder a des bibliotheques de code. System donne acces aux types de base comme Action<T> (les evenements). System.Collections.Generic donne acces aux collections typees comme HashSet<T>.

Remarque : les anciens imports inutiles (System.Linq, System.Text, System.Threading.Tasks) ont ete supprimes — on ne garde que ce qui est utilise.

CONCEPT Classe statique (L4)

Une static class est comme un tableau d'affichage commun dans un batiment : tout le monde peut y lire et ecrire, mais il n'existe qu'en un seul exemplaire. On ne peut pas creer d'instance (new GameProgress() serait interdit).

Pourquoi static ici ? La progression du jeu est un etat global unique — il n'y a qu'une seule partie en cours. N'importe quel systeme (WorldMap, DialogueNPC) peut verifier ou modifier la progression sans avoir besoin d'une reference vers un objet.

Avant ce commit, GameProgress n'etait pas static et contenait des dictionnaires vides. La transformation en static class simplifie enormement l'acces depuis partout dans le code.

CONCEPT const, HashSet et target-typed new (L6-8)

public const string PATH_LACORSE_NORTH definit une constante : une valeur fixee a la compilation qui ne changera jamais. C'est comme un numero de serie grave dans le metal. Utiliser des constantes nommees evite les "magic strings" eparpillees dans le code.

HashSet<string> est comme un sac de jetons uniques : on peut y ajouter un jeton, verifier s'il est present, mais impossible d'avoir deux fois le meme. Contrairement a une List, la recherche est en O(1) — quasi-instantanee quelle que soit la taille.

new() est le target-typed new (C# 9+) : le compilateur deduit le type depuis la declaration a gauche. new() equivaut ici a new HashSet<string>(). C'est plus concis sans perdre en clarte.

CONCEPT event et Action<string> (L10)

Un event est comme une liste de diffusion email : quand quelque chose se produit, tous les abonnes sont notifies automatiquement. Action<string> definit le "format du message" — ici, un string (le nom du flag).

OnFlagUnlocked est l'evenement declenche quand un flag est deverrouille. D'autres systemes peuvent s'y abonner avec GameProgress.OnFlagUnlocked += MaMethode; pour reagir aux deverrouillages.

Actuellement cet evenement n'est pas utilise dans le code de cette session, mais il est prevu pour des extensions futures (ex: mettre a jour la carte en temps reel).

LOGIQUE Methode Unlock — deverrouillage (L12-18)

La methode Unlock est le coeur du systeme. Elle ajoute un flag au HashSet.

Astuce subtile : HashSet.Add() retourne true si l'element n'existait pas encore, false s'il etait deja present. Le if autour evite donc de declencher l'evenement en double si on essaie de deverrouiller quelque chose deja deverrouille.

L'operateur ?. (null-conditional) sur OnFlagUnlocked?.Invoke(flag) verifie si quelqu'un est abonne a l'evenement avant de le declencher. Sans le ?., appeler Invoke sur un evenement sans abonne provoquerait une NullReferenceException.

LOGIQUE Methode IsUnlocked — verification (L20-23)

IsUnlocked est une simple question : "est-ce que ce flag est dans le sac ?". HashSet.Contains() repond en O(1).

C'est cette methode qui est appelee par WorldMap.IsNodeLocked() pour savoir si un chemin ou un bouton doit etre affiche ou cache.

ToastUI.cs NOUVEAU

Composant d'interface qui affiche des notifications temporaires (toasts) a l'ecran. Le message apparait en fondu, reste affiche quelques secondes, puis disparait. Utilise le pattern Singleton pour etre appele depuis n'importe ou via ToastUI.Show("message").

src/ui/ToastUI.cs 
using Godot;

public partial class ToastUI : Control
{
	static ToastUI instance;

	PanelContainer panel;
	Label label;

	public override void _Ready()
	{
		instance = this;

		// Build UI programmatically
		panel = new PanelContainer();
		panel.AnchorLeft = 0.5f;
		panel.AnchorRight = 0.5f;
		panel.AnchorTop = 0f;
		panel.GrowHorizontal = Control.GrowDirection.Both;
		panel.OffsetTop = 30;
		panel.OffsetLeft = -200;
		panel.OffsetRight = 200;

		// Style: dark semi-transparent background with golden border
		var styleBox = new StyleBoxFlat();
		styleBox.BgColor = new Color(0.1f, 0.1f, 0.12f, 0.85f);
		styleBox.BorderColor = new Color(0.85f, 0.65f, 0.2f, 1f);
		styleBox.SetBorderWidthAll(2);
		styleBox.SetCornerRadiusAll(6);
		styleBox.SetContentMarginAll(16);
		panel.AddThemeStyleboxOverride("panel", styleBox);

		// Label
		label = new Label();
		label.HorizontalAlignment = HorizontalAlignment.Center;
		label.AddThemeFontSizeOverride("font_size", 22);

		// Load Sylfean font
		var font = GD.Load<Font>("res://fonts/Sylfean.tres");
		if (font != null)
			label.AddThemeFontOverride("font", font);

		label.AddThemeColorOverride("font_color", new Color(0.95f, 0.85f, 0.55f));

		panel.AddChild(label);
		AddChild(panel);

		panel.Modulate = new Color(1, 1, 1, 0);
		panel.Visible = false;
	}

	public static void Show(string message, float duration = 3f)
	{
		instance?.DoShow(message, duration);
	}

	void DoShow(string message, float duration)
	{
		label.Text = message;
		panel.Visible = true;

		var tween = CreateTween();

		// Fade in
		tween.TweenProperty(panel, "modulate:a", 1f, 0.2f);

		// Wait
		tween.TweenInterval(duration);

		// Fade out
		tween.TweenProperty(panel, "modulate:a", 0f, 0.5f);

		tween.Finished += () => panel.Visible = false;
	}
}

Annotations

GODOT partial class et Control (L1-3)

partial class est obligatoire pour toute classe C# dans Godot 4. Le source generator de Godot genere automatiquement une autre partie de la classe (dans un fichier cache) pour connecter le code C# au moteur.

Control est la classe de base pour tous les elements d'interface (UI) dans Godot. En heritant de Control, notre ToastUI peut etre place dans un CanvasLayer et s'afficher par-dessus le jeu.

PATTERN Singleton (L5, L12, L52-55)

Le pattern Singleton garantit qu'une seule instance existe. C'est comme un haut-parleur unique dans un batiment : il n'y en a qu'un, et tout le monde peut l'utiliser sans savoir ou il est installe.

static ToastUI instance stocke la reference unique. Dans _Ready(), instance = this enregistre le noeud comme l'unique instance. La methode Show() utilise instance?.DoShow() pour deleguer a cette instance.

Pourquoi ici ? On veut pouvoir appeler ToastUI.Show("message") depuis n'importe quel script sans avoir a chercher le noeud dans l'arbre. C'est pratique mais attention : ce pattern cree un couplage global.

GODOT Construction UI programmatique dans _Ready (L10-50)

_Ready() est appele une seule fois quand le noeud entre dans l'arbre de scene. C'est le constructeur cote Godot.

Ici, toute l'interface est creee par code au lieu d'utiliser l'editeur visuel. PanelContainer est le conteneur de fond, et les proprietes Anchor* le positionnent au centre horizontal en haut de l'ecran. GrowDirection.Both permet au panneau de s'etendre dans les deux sens depuis son ancre.

Pourquoi par code ? Pour un element simple et reutilisable comme un toast, le creer par code evite de dependre d'une scene .tscn separee. Tout est autonome dans un seul fichier.

GODOT StyleBoxFlat — theming (L25-31)

StyleBoxFlat est le systeme de personnalisation visuelle de Godot. C'est comme une feuille CSS pour un element : on definit la couleur de fond, les bordures, les coins arrondis et les marges.

AddThemeStyleboxOverride("panel", styleBox) applique ce style au PanelContainer. Le "panel" est le nom du theme slot du PanelContainer dans Godot.

Les couleurs choisies (fond sombre semi-transparent, bordure doree) correspondent au theme medieval du jeu.

GODOT GD.Load et chargement de ressources (L39-41)

GD.Load<Font>("res://...") charge une ressource depuis le systeme de fichiers du projet Godot. Le prefixe res:// pointe vers la racine du projet.

Le if (font != null) est une securite : si la police n'existe pas, on garde la police par defaut au lieu de crasher. C'est une bonne pratique pour les ressources optionnelles.

GODOT Modulate et transparence (L48-49)

Modulate est un multiplicateur de couleur applique au noeud et tous ses enfants. new Color(1, 1, 1, 0) signifie : couleurs normales (R=1, G=1, B=1) mais completement transparent (A=0).

Le panneau est invisible au demarrage (Visible = false + alpha a 0). Quand un toast est affiche, on anime l'alpha de 0 a 1 pour un effet de fondu.

PATTERN Facade statique (L52-55)

La methode public static void Show() est une facade : elle offre une interface simple pour une operation complexe. L'appelant ecrit juste ToastUI.Show("Chemin deverrouille !") sans se soucier du panel, du label, du tween, etc.

instance?.DoShow() utilise le null-conditional ?. pour eviter un crash si aucune instance n'existe (par exemple si le noeud ToastUI n'a pas ete ajoute a la scene).

Le parametre duration = 3f est une valeur par defaut : si on ne precise pas la duree, le toast reste 3 secondes.

GODOT Tween — animation procedurale (L62-73)

Un Tween est un animateur automatique : on lui dit "change cette propriete de A a B en N secondes" et il gere l'interpolation. C'est comme un pilote automatique pour les animations.

CreateTween() cree un nouveau tween attache au noeud actuel. Les appels sont chainables et sequentiels :

  • TweenProperty(panel, "modulate:a", 1f, 0.2f) → anime l'alpha de 0 a 1 en 0.2s (fondu entrant)
  • TweenInterval(duration) → attend N secondes (pause)
  • TweenProperty(panel, "modulate:a", 0f, 0.5f) → anime l'alpha de 1 a 0 en 0.5s (fondu sortant)

tween.Finished += () => panel.Visible = false utilise une expression lambda pour cacher le panneau a la fin de l'animation. C'est un callback anonyme, plus concis qu'une methode nommee pour une action aussi simple.

DialogueNPC.cs MODIFIE

Represente un NPC interactif qui peut declencher un dialogue. Les modifications ajoutent : un flag de deverrouillage (UnlockFlag), un dialogue alternatif apres la premiere interaction, et un callback qui deverrouille un chemin + affiche un toast quand le dialogue se termine.

src/dialogue/DialogueNPC.cs 
using Godot;

public partial class DialogueNPC : StaticBody3D, IInteractable
{
	[Export]
	public DialogueDefinition Dialogue;

	[Export]
	public string NPCName = "NPC";

	[Export]
	public string UnlockFlag = "";

	[Export]
	public DialogueDefinition DialogueAfterUnlock;

	Area3D interactZone;
	Label3D promptLabel;
	Label3D nameLabel;
	bool playerInRange;
	bool waitingForDialogueEnd;

	string DoneFlagName => $"npc_done_{NPCName}";

	public override void _Ready()
	{
		InteractManager.interactableList.Add(this);

		interactZone = GetNode<Area3D>("InteractZone");
		promptLabel = GetNode<Label3D>("PromptLabel");
		nameLabel = GetNode<Label3D>("NameLabel");

		promptLabel.Visible = false;
		nameLabel.Text = NPCName;

		interactZone.BodyEntered += OnBodyEntered;
		interactZone.BodyExited += OnBodyExited;

		// Connect to DialogueManager's finished signal if we need to track dialogue completion
		if (!string.IsNullOrEmpty(UnlockFlag) || DialogueAfterUnlock != null)
		{
			var dm = GetTree().CurrentScene.FindChild("DialogueManager", true, false) as DialogueManager;
			if (dm != null)
				dm.DialogueFinished += OnDialogueFinished;
		}
	}

	void OnBodyEntered(Node3D body)
	{
		if (body is CampCharacter)
		{
			playerInRange = true;
			promptLabel.Visible = true;
		}
	}

	void OnBodyExited(Node3D body)
	{
		if (body is CampCharacter)
		{
			playerInRange = false;
			promptLabel.Visible = false;
		}
	}

	public bool TryInteract()
	{
		if (playerInRange && !DialogueManager.IsDialogueActive)
		{
			// Use alternate dialogue if NPC already talked to
			bool alreadyDone = DialogueAfterUnlock != null && GameProgress.IsUnlocked(DoneFlagName);
			var dialogueToUse = alreadyDone ? DialogueAfterUnlock : Dialogue;

			DialogueManager.StartDialogue(dialogueToUse);

			if (!alreadyDone)
				waitingForDialogueEnd = true;

			return true;
		}
		return false;
	}

	void OnDialogueFinished()
	{
		if (!waitingForDialogueEnd) return;
		waitingForDialogueEnd = false;

		// Unlock path if needed
		if (!string.IsNullOrEmpty(UnlockFlag) && !GameProgress.IsUnlocked(UnlockFlag))
		{
			GameProgress.Unlock(UnlockFlag);
			ToastUI.Show("Chemin deverrouille !");
		}

		// Mark NPC as talked to
		if (DialogueAfterUnlock != null)
			GameProgress.Unlock(DoneFlagName);
	}
}

Annotations

CONCEPT Interface IInteractable (L3)

Une interface est un contrat : toute classe qui l'implemente promet de fournir certaines methodes. C'est comme un badge "je suis interactif" que le NPC porte.

IInteractable oblige la classe a avoir une methode TryInteract(). Le systeme InteractManager peut ainsi traiter tous les objets interactifs de maniere uniforme, qu'il s'agisse d'un NPC, d'un coffre ou d'une porte.

StaticBody3D est le corps physique du NPC dans le monde 3D — il ne bouge pas mais peut etre detecte par les collisions.

GODOT Champs [Export] (L5-15)

L'attribut [Export] est comme une etiquette "configurable" : il expose le champ dans l'inspecteur de l'editeur Godot. Un game designer peut ainsi configurer chaque NPC sans toucher au code.

Nouveaux champs ajoutes :

  • UnlockFlag — le nom du flag a deverrouiller apres le dialogue (ex: "path_lacorse_north"). Vide = pas de deverrouillage.
  • DialogueAfterUnlock — un dialogue alternatif joue si le joueur reparle au NPC apres l'avoir deja rencontre.

Ces champs permettent de configurer le comportement du NPC entierement depuis l'editeur : quel dialogue jouer, quel chemin deverrouiller, et quel dialogue afficher ensuite.

CONCEPT Expression-bodied property (L23)

string DoneFlagName => $"npc_done_{NPCName}" est une propriete calculee ecrite en forme compacte. A chaque lecture, elle retourne la valeur calculee. C'est equivalent a :

string DoneFlagName { get { return $"npc_done_{NPCName}"; } }

Le $"..." est une string interpolee : les expressions entre {} sont evaluees et inserees dans la chaine. Si NPCName vaut "Marchand", DoneFlagName retourne "npc_done_Marchand".

Ce flag sert a marquer qu'on a deja parle a ce NPC specifique, pour savoir s'il faut jouer le dialogue initial ou le dialogue alternatif.

GODOT Arbre de scene et connexion d'evenements (L25-45)

Dans _Ready(), le NPC se configure :

  • GetNode<Area3D>("InteractZone") cherche un noeud enfant par son nom. Le <Area3D> est un cast generique : "je veux ce noeud ET je m'attends a ce qu'il soit un Area3D".
  • BodyEntered / BodyExited sont des signaux Godot emis quand un corps entre/sort de la zone de detection.

Nouveau (L39-45) : Si le NPC a un flag de deverrouillage ou un dialogue alternatif, il cherche le DialogueManager dans la scene et s'abonne a son evenement DialogueFinished.

GetTree().CurrentScene.FindChild("DialogueManager", true, false) parcourt l'arbre de scene pour trouver le noeud. Le as DialogueManager est un cast securise : il retourne null au lieu de crasher si le noeud n'est pas du bon type.

GODOT Detection de proximite avec Area3D (L48-64)

Area3D est une zone invisible dans le monde 3D qui detecte les objets qui y entrent/sortent. C'est comme un capteur de mouvement a l'entree d'un magasin.

body is CampCharacter est du pattern matching C# : il verifie si le corps qui entre est un joueur (CampCharacter) et non un autre NPC ou objet. Seul le joueur declenche l'affichage du prompt d'interaction.

LOGIQUE TryInteract avec dialogue alternatif (L66-82)

La methode TryInteract() est appelee par le systeme d'interaction quand le joueur presse la touche d'interaction.

Logique modifiee :

  1. Verifier que le joueur est a portee ET qu'aucun dialogue n'est actif
  2. Nouveau : Verifier si le NPC a deja ete rencontre (GameProgress.IsUnlocked(DoneFlagName))
  3. Si oui et qu'un dialogue alternatif existe → jouer le dialogue alternatif
  4. Sinon → jouer le dialogue principal et marquer qu'on attend la fin

L'operateur ternaire alreadyDone ? DialogueAfterUnlock : Dialogue est un raccourci pour un if/else : "si deja fait, utilise l'alternatif, sinon l'original".

LOGIQUE OnDialogueFinished — le coeur du deverrouillage (L84-99)

C'est la methode la plus importante de cette session. Elle est appelee quand un dialogue se termine et orchestre tout le processus de deverrouillage :

  1. Guard clause (L86-87) : si on n'attend pas la fin d'un dialogue, on ne fait rien. Cela evite de reagir aux dialogues des autres NPCs.
  2. Deverrouillage (L90-93) : si un UnlockFlag est configure et pas encore deverrouille → appel de GameProgress.Unlock() + affichage du toast.
  3. Marquage (L97-98) : marque le NPC comme "deja rencontre" pour que la prochaine interaction utilise le dialogue alternatif.

C'est ici que les trois systemes se connectent : DialogueNPC → GameProgress → ToastUI.

WorldMap.cs MODIFIE

Gere la carte du monde interactive. Le joueur se deplace entre des boutons relies par des lignes. Les modifications ajoutent : filtrage des noeuds/chemins verrouilles via GameProgress, un algorithme BFS d'accessibilite pour masquer les zones isolees, et la sauvegarde de la position du joueur.

src/core/WorldMap.cs 
using Godot;
using System.Collections.Generic;
using System.Linq;
using tactics_game_proto.ui.world_map;

public partial class WorldMap : Node
{
	private static string savedNodeName;

	private Node2D mapRoot;
	private Sprite2D character;
	private WorldMapBtn currentNode;
	private bool isMoving;

	// Adjacency: for each button, list of (neighbor, path points ordered from this button toward neighbor)
	private Dictionary<WorldMapBtn, List<(WorldMapBtn neighbor, Vector2[] points)>> adjacency = new();

	public override void _Ready()
	{
		mapRoot = GetNode<Node2D>("Holder/Draggable_Control/map_root");
		var linksNode = mapRoot.GetNode<Node2D>("links");

		// Collect all BtnBattleStarter children, filtering locked ones
		var buttons = new List<WorldMapBtn>();
		foreach (var child in mapRoot.GetChildren())
		{
			if (child is WorldMapBtn btn)
			{
				if (IsNodeLocked(btn))
				{
					btn.Visible = false;
					continue;
				}
				buttons.Add(btn);
			}
		}

		// Initialize adjacency lists
		foreach (var btn in buttons)
			adjacency[btn] = new List<(WorldMapBtn, Vector2[])>();

		// Build graph from Line2D connections, filtering locked lines
		foreach (var child in linksNode.GetChildren())
		{
			if (child is not Line2D line || line.Points.Length < 2)
				continue;

			if (IsNodeLocked(line))
			{
				line.Visible = false;
				continue;
			}

			var points = line.Points;
			var lineOffset = line.Position;

			// Convert Line2D local points to map_root coordinates
			var worldPoints = new Vector2[points.Length];
			for (int i = 0; i < points.Length; i++)
				worldPoints[i] = points[i] + lineOffset;

			// Find nearest buttons for first and last points
			var btnA = FindNearestButton(buttons, worldPoints[0]);
			var btnB = FindNearestButton(buttons, worldPoints[^1]);

			if (btnA == null || btnB == null || btnA == btnB)
				continue;

			// Store both directions (forward and reversed)
			adjacency[btnA].Add((btnB, worldPoints));
			adjacency[btnB].Add((btnA, worldPoints.Reverse().ToArray()));
		}

		// Reachability pass: hide buttons and lines not reachable from starting node
		var startBtn = buttons.FirstOrDefault(b => b.Name == "btn_battle") ?? buttons[0];
		var reachable = FindReachableButtons(startBtn);

		// Hide unreachable buttons and remove them from graph
		foreach (var btn in buttons.Where(b => !reachable.Contains(b)).ToList())
		{
			btn.Visible = false;
			adjacency.Remove(btn);
			buttons.Remove(btn);
		}

		// Remove edges pointing to unreachable buttons
		foreach (var btn in buttons)
			adjacency[btn].RemoveAll(e => !reachable.Contains(e.neighbor));

		// Hide Line2D whose endpoints don't both resolve to reachable buttons
		foreach (var child in linksNode.GetChildren())
		{
			if (child is not Line2D line || !line.Visible)
				continue;

			var pts = line.Points;
			var offset = line.Position;
			var a = FindNearestButton(buttons, pts[0] + offset);
			var b = FindNearestButton(buttons, pts[^1] + offset);

			if (a == null || b == null)
				line.Visible = false;
		}

		// Create the player character sprite
		CreateCharacter();

		// Restore saved position or default to start
		if (savedNodeName != null)
			currentNode = buttons.FirstOrDefault(b => b.Name == savedNodeName) ?? startBtn;
		else
			currentNode = startBtn;
		character.Position = GetButtonCenter(currentNode);

		// Intercept button clicks
		foreach (var btn in buttons)
		{
			btn.ManagedByWorldMap = true;
			var captured = btn;
			captured.Pressed += () => OnBattleButtonPressed(captured);
		}
	}

	private void CreateCharacter()
	{
		var image = Image.CreateEmpty(24, 24, false, Image.Format.Rgba8);
		image.Fill(new Color(0.2f, 0.4f, 0.9f));
		var texture = ImageTexture.CreateFromImage(image);

		character = new Sprite2D();
		character.Texture = texture;
		character.ZIndex = 1;
		mapRoot.AddChild(character);
	}

	private Vector2 GetButtonCenter(WorldMapBtn btn)
	{
		return btn.Position + btn.Size / 2;
	}

	private WorldMapBtn FindNearestButton(List<WorldMapBtn> buttons, Vector2 point)
	{
		WorldMapBtn nearest = null;
		float minDist = float.MaxValue;

		foreach (var btn in buttons)
		{
			float dist = GetButtonCenter(btn).DistanceTo(point);
			if (dist < minDist)
			{
				minDist = dist;
				nearest = btn;
			}
		}

		return minDist < 50f ? nearest : null;
	}

	private void OnBattleButtonPressed(WorldMapBtn btn)
	{
		if (isMoving)
			return;

		if (btn == currentNode)
		{
			btn.OnTrigerred();
			return;
		}

		var path = FindPath(currentNode, btn);
		if (path == null || path.Count < 2)
			return;

		MoveAlongPath(path);
	}

	/// <summary>
	/// BFS pathfinding from one node to another.
	/// </summary>
	private List<WorldMapBtn> FindPath(WorldMapBtn from, WorldMapBtn to)
	{
		var visited = new HashSet<WorldMapBtn> { from };
		var queue = new Queue<List<WorldMapBtn>>();
		queue.Enqueue(new List<WorldMapBtn> { from });

		while (queue.Count > 0)
		{
			var current = queue.Dequeue();
			var last = current[^1];

			if (last == to)
				return current;

			foreach (var (neighbor, _) in adjacency[last])
			{
				if (visited.Contains(neighbor))
					continue;

				visited.Add(neighbor);
				var newPath = new List<WorldMapBtn>(current) { neighbor };
				queue.Enqueue(newPath);
			}
		}

		GD.PushWarning($"WorldMap: No path found from {from.Name} to {to.Name}");
		return null;
	}

	/// <summary>
	/// Moves the character along the BFS path using a Tween.
	/// </summary>
	private void MoveAlongPath(List<WorldMapBtn> nodePath)
	{
		isMoving = true;

		// Build waypoints from Line2D path data
		var waypoints = new List<Vector2>();

		for (int i = 0; i < nodePath.Count - 1; i++)
		{
			var from = nodePath[i];
			var to = nodePath[i + 1];

			var edge = adjacency[from].FirstOrDefault(e => e.neighbor == to);
			if (edge.points == null)
				continue;

			// Skip first point of each segment (it's where we already are or the junction just reached)
			for (int j = 1; j < edge.points.Length; j++)
				waypoints.Add(edge.points[j]);
		}

		if (waypoints.Count == 0)
		{
			isMoving = false;
			return;
		}

		// Snap last waypoint to exact button center
		waypoints[^1] = GetButtonCenter(nodePath[^1]);

		// Create tween chain
		var targetNode = nodePath[^1];
		var tween = CreateTween();
		const float speed = 300f; // pixels per second

		var prevPos = character.Position;
		foreach (var wp in waypoints)
		{
			float dist = prevPos.DistanceTo(wp);
			double duration = dist / speed;
			if (duration < 0.01)
				continue;

			tween.TweenProperty(character, "position", wp, duration);
			prevPos = wp;
		}

		tween.Finished += () =>
		{
			currentNode = targetNode;
			savedNodeName = currentNode.Name;
			isMoving = false;
		};
	}

	{
	}

	/// <summary>
	/// BFS from a starting button to find all reachable buttons in the graph.
	/// </summary>
	private HashSet<WorldMapBtn> FindReachableButtons(WorldMapBtn start)
	{
		var visited = new HashSet<WorldMapBtn> { start };
		var queue = new Queue<WorldMapBtn>();
		queue.Enqueue(start);

		while (queue.Count > 0)
		{
			var current = queue.Dequeue();
			if (!adjacency.ContainsKey(current)) continue;

			foreach (var (neighbor, _) in adjacency[current])
			{
				if (visited.Add(neighbor))
					queue.Enqueue(neighbor);
			}
		}

		return visited;
	}

	/// <summary>
	/// Checks if a node has a metadata unlock_flag that hasn't been unlocked yet.
	/// </summary>
	private bool IsNodeLocked(Node node)
	{
		if (!node.HasMeta("unlock_flag"))
			return false;

		string flag = node.GetMeta("unlock_flag").AsString();
		return !string.IsNullOrEmpty(flag) && !GameProgress.IsUnlocked(flag);
	}
}

Annotations

CONCEPT Variable static pour persistance inter-scene (L8)

private static string savedNodeName — Le mot-cle static fait que cette variable survit aux changements de scene. Quand le joueur quitte la carte pour un combat puis revient, la position est memorisee.

C'est comme une note collante sur le bureau : meme si on range tous les papiers (destruction de la scene), la note reste. Attention : static est perdu si le jeu redemarre (pas de sauvegarde disque ici).

CONCEPT Dictionary, ValueTuple et graphe d'adjacence (L16)

La ligne Dictionary<WorldMapBtn, List<(WorldMapBtn neighbor, Vector2[] points)>> est dense mais logique :

  • Dictionary<K,V> = table de correspondance. La cle est un bouton, la valeur est la liste de ses voisins.
  • (WorldMapBtn neighbor, Vector2[] points) est un ValueTuple nomme : un regroupement de deux valeurs (le voisin + les points du chemin). C'est comme un post-it avec deux infos.

Au final, cette structure represente un graphe d'adjacence : pour chaque bouton, on connait tous les boutons voisins et les coordonnees des chemins qui y menent. C'est la colonne vertebrale de la navigation sur la carte.

LOGIQUE Filtrage des noeuds verrouilles (L23-36, L48-52)

C'est le coeur de la feature de cette session. Lors de la construction de la carte, chaque bouton et chaque ligne est verifie :

  1. Pour les boutons (L29-33) : si IsNodeLocked(btn) est vrai → on cache le bouton et on passe au suivant (continue)
  2. Pour les lignes (L48-52) : meme logique, les lignes vers des noeuds verrouilles sont masquees

Le filtre s'applique au moment du _Ready(), donc a chaque fois que la scene de la carte est chargee. Si un flag a ete deverrouille depuis la derniere visite, les elements correspondants apparaitront.

LOGIQUE Construction du graphe depuis les Line2D (L42-72)

La carte du monde est modelisee dans l'editeur Godot avec des Line2D (lignes visuelles) qui relient des WorldMapBtn (boutons cliquables). Le code transforme cette representation visuelle en graphe manipulable :

  1. Pour chaque Line2D, on recupere les points et on les convertit en coordonnees globales (L58-60)
  2. On cherche les boutons les plus proches du debut et de la fin de la ligne (L63-64)
  3. On ajoute les connexions dans les deux sens (L70-71) — le graphe est bidirectionnel

worldPoints.Reverse().ToArray() utilise LINQ pour inverser le tableau de points quand on stocke la direction opposee.

worldPoints[^1] utilise l'index from end (C# 8+) : ^1 signifie "le dernier element". C'est plus lisible que worldPoints[worldPoints.Length - 1].

LOGIQUE BFS d'accessibilite (L74-103)

Probleme : Apres avoir masque les noeuds verrouilles, certains boutons peuvent se retrouver isoles — plus aucun chemin ne mene a eux depuis le point de depart. Il faut les masquer aussi.

Solution : Un BFS (Breadth-First Search) depuis le noeud de depart. C'est comme une inondation : on part du point de depart et on "inonde" chaque voisin accessible. Tout ce qui n'est pas atteint par l'eau est isole.

  1. FindReachableButtons(startBtn) retourne l'ensemble des boutons atteignables
  2. Les boutons non-atteignables sont caches et retires du graphe (L79-83)
  3. Les aretes vers les boutons supprimes sont nettoyees (L86-88)
  4. Les lignes visuelles correspondantes sont cachees (L91-103)

buttons.Where(b => !reachable.Contains(b)).ToList() — le .ToList() est crucial : il materialise la liste AVANT de modifier buttons dans la boucle. Sans lui, on aurait une erreur "collection modified during iteration".

LOGIQUE Position sauvegardee et restoration (L105-121)

Quand le joueur revient sur la carte apres un combat, il doit retrouver sa position. Le champ static savedNodeName retient le nom du dernier noeud visite.

buttons.FirstOrDefault(b => b.Name == savedNodeName) ?? startBtn :

  • FirstOrDefault (LINQ) : cherche le premier bouton dont le nom correspond. Retourne null si aucun ne correspond.
  • ?? (null-coalescing) : si le resultat est null → utilise le bouton de depart par defaut.

Capture de variable (L119-120) : var captured = btn puis captured.Pressed += () => .... C'est necessaire parce que la variable btn de la boucle foreach est la meme reference a chaque iteration. Sans la capture, toutes les lambdas utiliseraient le dernier bouton.

LOGIQUE Recherche spatiale — FindNearestButton (L141-157)

Cette methode cherche le bouton le plus proche d'un point donne, avec un seuil de 50 pixels. C'est un simple parcours lineaire de tous les boutons.

float.MaxValue est utilise comme valeur initiale pour la distance minimale — n'importe quel bouton reel sera plus proche. Le seuil de 50 pixels evite de connecter des lignes a des boutons trop eloignes.

LOGIQUE BFS pathfinding — FindPath (L180-207)

L'algorithme BFS (Breadth-First Search) trouve le chemin le plus court entre deux boutons dans le graphe. C'est comme explorer un labyrinthe couche par couche : d'abord les voisins directs, puis les voisins des voisins, etc.

Le BFS garantit de trouver le chemin le plus court en nombre de noeuds. La file (Queue) assure que les chemins courts sont explores avant les longs.

var (neighbor, _) est une deconstruction de tuple avec discard : on extrait le voisin mais on ignore les points du chemin (note par _).

GODOT Chaine de Tweens pour le deplacement (L212-265)

Le deplacement du joueur suit les points des Line2D segment par segment. Pour chaque waypoint, un TweenProperty anime la position du sprite.

La vitesse est constante (300 px/s) et la duree de chaque segment est calculee par distance / vitesse. Les segments trop courts (< 0.01s) sont ignores pour eviter des micro-animations.

A la fin du mouvement, savedNodeName = currentNode.Name memorise la position pour la persistence entre scenes.

LOGIQUE IsNodeLocked — pont vers GameProgress (L298-305)

Cette methode est le pont entre le systeme visuel (WorldMap) et le systeme de progression (GameProgress) :

  1. node.HasMeta("unlock_flag") — verifie si le noeud Godot a une metadata nommee "unlock_flag". Les metadata sont des donnees personnalisees attachees aux noeuds dans l'editeur.
  2. node.GetMeta("unlock_flag").AsString() — recupere la valeur et la convertit en string
  3. GameProgress.IsUnlocked(flag) — verifie si ce flag a ete deverrouille

Si le flag n'existe pas dans GameProgress, le noeud est considere comme verrouille. Le game designer n'a qu'a ajouter une metadata "unlock_flag" sur un bouton ou une ligne dans l'editeur pour le rendre conditionnel.

CONCEPT Enum GameState (L9-18)

Un enum est une liste fermee de choix possibles. C'est comme un panneau indicateur avec un nombre fixe de directions. Le jeu ne peut etre que dans UN de ces etats a la fois : menu principal, carte du monde, camp, combat, ou exploration.

Utiliser un enum au lieu de strings evite les fautes de frappe et permet au compilateur de verifier les valeurs. GameState.None est l'etat initial avant le premier chargement.

PATTERN Singleton + [Export] Dictionary (L22-38)

Comme ToastUI, Game utilise un Singleton (static Game self) pour etre accessible globalement via les methodes statiques.

[Export] Dictionary<GameState, PackedScene> gameStatesAssets est particulierement elegant : c'est un dictionnaire configurable dans l'editeur. Le game designer associe chaque etat (WorldMap, Camp, Battle) a une scene .tscn. Le code n'a plus besoin de paths en dur.

Changement de cette session : Le champ GameProgress progress qui existait avant a ete supprime. Puisque GameProgress est maintenant une static class, il n'a plus besoin d'etre instancie ou reference.

LOGIQUE Changement d'etat — RequestChangeGameState (L88-111)

Cette methode est le chef d'orchestre du jeu. Quand on veut changer d'etat (ex: de WorldMap vers Battle) :

  1. Verifie qu'on ne change pas vers le meme etat (guard clause)
  2. Instancie la nouvelle scene depuis le dictionnaire (Instantiate<Node>())
  3. Detruit l'ancienne scene (QueueFree() — destruction differee, plus sure)
  4. Ajoute la nouvelle scene a l'arbre
  5. Notifie les abonnes via onGameStateChanged?.Invoke()

QueueFree() vs Free() : QueueFree attend la fin du frame actuel pour detruire le noeud, evitant les crashs si le noeud est encore en cours d'utilisation.

Glossaire

Tous les termes techniques utilises dans ce document, expliques simplement.

static class
Classe qui ne peut pas etre instanciee. Tous ses membres sont partages globalement. Accessible depuis n'importe ou sans creer d'objet.
partial class
Classe dont le code est reparti dans plusieurs fichiers. Obligatoire en Godot 4 C# car le moteur genere automatiquement du code complementaire.
HashSet<T>
Collection d'elements uniques. L'ajout d'un doublon est ignore. Recherche en O(1) — quasi-instantanee quelle que soit la taille.
Dictionary<K,V>
Table de correspondance cle/valeur. Chaque cle est unique et pointe vers une valeur. Recherche en O(1).
List<T>
Collection dynamique ordonnee. On peut ajouter, supprimer, acceder par index. Taille variable.
Queue<T>
File d'attente FIFO (First In, First Out). Le premier element ajoute est le premier retire. Utilisee dans les algorithmes BFS.
ValueTuple
Regroupement leger de valeurs avec noms optionnels. (string name, int age) est un tuple de deux elements.
event / Action<T>
Systeme de notification. Un event peut avoir des abonnes qui sont prevenus quand l'event est declenche. Action<string> est un delegate qui prend un string en parametre.
delegate
Reference vers une methode. Action<T> est un delegate predefined qui ne retourne rien. Permet de passer des methodes comme parametres.
Expression lambda
Fonction anonyme compacte. () => expression (sans parametres) ou x => expression (un parametre). Souvent utilisee pour les callbacks et LINQ.
Operateur ?. (null-conditional)
Appel securise. obj?.Method() n'appelle Method que si obj n'est pas null. Evite les NullReferenceException.
Operateur ?? (null-coalescing)
Valeur par defaut. x ?? y retourne x si non-null, sinon y.
Pattern matching (is / is not)
Test de type elegant. obj is MyType t verifie le type et cree une variable typee en une seule operation.
Index from end (^N)
Acces depuis la fin d'une collection. [^1] = dernier element, [^2] = avant-dernier. C# 8+.
Target-typed new
Raccourci d'initialisation. List<int> x = new() au lieu de new List<int>(). Le type est deduit du contexte. C# 9+.
String interpolation
Construction de chaines avec $"texte {variable}". Les expressions entre accolades sont evaluees et inserees.
Expression-bodied member
Syntaxe compacte avec =>. string Name => "valeur" est equivalent a une propriete avec un getter.
Operateur ternaire (? :)
Raccourci if/else en une expression. condition ? siVrai : siFaux.
LINQ
Language Integrated Query. Methodes d'extension pour filtrer, trier, transformer des collections. .Where(), .FirstOrDefault(), .Select(), etc.
enum
Type a valeurs nommees fixes. Ex: enum Couleur { Rouge, Vert, Bleu }. Le compilateur garantit qu'on ne peut utiliser que les valeurs definies.
const
Constante evaluee a la compilation. La valeur est fixee dans le code et ne peut jamais changer.
readonly
Champ qui ne peut etre modifie qu'a l'initialisation ou dans le constructeur. Plus flexible que const car la valeur peut etre calculee.
interface
Contrat definissant un ensemble de methodes qu'une classe doit implementer. Permet le polymorphisme sans heritage.
BFS (Breadth-First Search)
Algorithme de parcours en largeur d'un graphe. Explore tous les voisins directs avant de passer aux voisins des voisins. Garantit le chemin le plus court en nombre d'etapes.
Singleton
Design pattern garantissant qu'une seule instance d'une classe existe. Offre un point d'acces global.
Tween
Outil d'animation procedurale de Godot. Interpole une propriete d'une valeur a une autre sur une duree donnee.
[Export]
Attribut Godot qui expose un champ dans l'inspecteur de l'editeur. Permet de configurer les valeurs sans modifier le code.
_Ready()
Methode du cycle de vie Godot, appelee une fois quand le noeud entre dans l'arbre de scene. Equivalent d'un constructeur cote moteur.
_Process(double delta)
Methode du cycle de vie Godot, appelee a chaque frame. delta est le temps ecoule depuis le dernier frame.
PackedScene / Instantiate
Modele de scene reutilisable. Instantiate() cree une copie independante de la scene en memoire.
QueueFree()
Suppression differee d'un noeud Godot. Attend la fin du frame actuel, evitant les problemes si le noeud est encore utilise.
Area3D
Zone de detection 3D dans Godot. Emet des signaux quand des corps entrent ou sortent de son volume.
Node metadata
Donnees personnalisees attachees a un noeud Godot via SetMeta()/GetMeta(). Configurables dans l'editeur ou par code.
