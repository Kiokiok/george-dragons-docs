Equipment System Documentation

Overview

The Equipment System allows pawns to equip items that modify ability behaviors, grant stat bonuses, and enhance combat capabilities. Equipment integrates seamlessly with the existing event pipeline to alter damage, healing, movement, and status effects without requiring ability code changes.

Core Components

BaseEquipment

Abstract base class for all equipment. Defines:

  • Id & Name - Unique identifier and display name
  • EquipmentSlot - Where the item can be equipped (Weapon, Armor, Helmet, Boots, Accessory1, Accessory2)
  • Lifecycle Methods:
    • OnEquip(pawn) - Called when equipped; subscribe to pawn events here
    • OnUnequip(pawn) - Called when unequipped; unsubscribe from events here

EquipmentController

Manages all equipped items for a pawn:

  • Equip(equipment) - Equips an item to its designated slot
  • Unequip(slot) - Removes equipped item from slot
  • GetEquipped(slot) - Retrieves equipped item or null
  • GetMovementPointBonus() - Sums MP bonuses from all equipped items
  • GetActionPointBonus() - Sums AP bonuses from all equipped items

Equipment Integration with Pawns

Equipment is managed via a pawn's EquipmentController:

BattlePawn.EquipmentController
├── GetMovementPointBonus() → Increases GetEffectiveMovementPoints()
├── GetActionPointBonus() → Applied when resetting AP in OnEndTurn()
└── Equipment events → Hook into pawn's damage/heal/movement events

How Equipment Alters Abilities

Equipment modifies ability behaviors through event subscriptions:

  1. Before Event - Equipment modifies the action packet before it's applied
  2. After Event - Equipment reacts to completed actions (add effects, heal, damage)

Examples:

  • DamageBoostWeapon - Subscribes to OnBeforeDealDamage, adds flat damage
  • RingOfPoison - Subscribes to OnAfterDealDamage, applies poison DoT to targets
  • HelmetOfProtection - Subscribes to OnBeforeTakeDamage, reduces incoming damage
  • StaffOfHealing - Subscribes to OnBeforeHealOther, amplifies healing output

Event Subscriptions

Equipment uses the same pawn event system as status effects:

Event Purpose Equipment Example
OnBeforeDealDamage Modify outgoing damage DamageBoostWeapon (+2 damage)
OnAfterDealDamage React to dealt damage RingOfPoison (add DoT)
OnBeforeTakeDamage Modify incoming damage HelmetOfProtection (-1 damage)
OnAfterTakeDamage React to taken damage ArmorOfReflection (damage attacker)
OnBeforeHealOther Modify outgoing healing StaffOfHealing (+2 healing)
OnBeforeMove Modify movement requests BootsOfSwiftness (increase MP)

Passive Bonuses

Equipment can grant permanent stat increases:

  • Movement Point Bonus - Added to effective MP pool
  • Action Point Bonus - Added when resetting AP at turn start

These bonuses aggregate from all equipped items and are applied automatically.

Usage Example

// Equip a weapon
DamageBoostWeapon sword = new DamageBoostWeapon();
pawn.EquipmentController.Equip(sword);

// All damage dealt now gets +2 automatically
// No ability code needs to change!

// Equip armor
HelmetOfProtection helmet = new HelmetOfProtection();
pawn.EquipmentController.Equip(helmet);

// All incoming damage is now reduced by 1

// Query bonuses
int totalMPBonus = pawn.EquipmentController.GetMovementPointBonus();
int totalAPBonus = pawn.EquipmentController.GetActionPointBonus();

Creating Custom Equipment

Extend BaseEquipment and override lifecycle methods:

public class MyCustomEquipment : BaseEquipment
{
    public override void OnEquip(BattlePawn pawn)
    {
        pawn.OnBeforeDealDamage += ModifyDamage;
    }

    public override void OnUnequip(BattlePawn pawn)
    {
        pawn.OnBeforeDealDamage -= ModifyDamage;
    }

    private void ModifyDamage(DamagePacket packet)
    {
        // Modify the packet as needed
        packet.FinalAmount += 3;
    }
}

Key Design Principles

✅ Non-Intrusive - Equipment modifies abilities through events, not by changing ability code
✅ Composable - Multiple equipment pieces stack effects through the same event system
✅ Automatic - Equip → Subscribe to events → Ability uses modified values → Unequip → Unsubscribe
✅ Type-Safe - Modifies DamagePackets, HealPackets, and other typed objects
✅ Efficient - Event-based system avoids unnecessary recalculations
✅ Extensible - New equipment types don't require changes to BattlePawn or BattleInstance

Equipment Slots

Equipment can occupy specific slots on a pawn. Only one item per slot:

  • Weapon - Primary attack tool
  • Armor - Body protection
  • Belt - ...
  • Cape - ...
  • Helmet - Head protection
  • Boots - Movement enhancement
  • Ring - ...
  • Amulet - ...

Equipping to an occupied slot automatically unequips the previous item.
