GroundCoverMeshGenerator

A procedural mesh generator for ground cover surfaces in a tile-based tactics game (Godot 4 / C#). It produces a single ArrayMesh covering arbitrary tile positions, complete with skirts (horizontal extensions on exposed edges) and corner patches to fill gaps.

Algorithm Overview

The generator takes a list of Vector3 tile positions (X, Y, Z = grid X, height, grid Z) and produces a mesh in several stages:

1 Build Lookup Structures

Create a HashSet<Vector2I> for O(1) neighbor existence checks, and a Dictionary<Vector2I, float> mapping each tile to its height.

HashSet<Vector2I> tileSet;
Dictionary<Vector2I, float> heightMap;

foreach (pos in tilePositions) {
    key = new Vector2I(pos.X, pos.Z);
    tileSet.Add(key);
    heightMap[key] = pos.Y;
}

2 Neighbor Detection

For each tile, check all 8 neighbors (4 cardinal + 4 diagonal). Cardinal neighbors determine skirt needs. Diagonal neighbors (height-aware) determine inner corner clipping.

// Cardinal: N, S, E, W
hasNorth = tileSet.Contains(northKey);
...
// Diagonal: NE, NW, SE, SW
hasNE = tileSet.Contains(neKey);
...
// Height-aware diagonals
hasNE_level = hasNE &&
    heightMap[neKey] >= h - 0.001f;

3 Skirt Decision Logic

An edge needs a skirt if: neighbor is absent, OR neighbor is lower. No skirt if neighbor is at the same height or above (the neighbor's cover mesh handles the transition).

northNeedsSkirt =
    !hasNorth ||
    heightMap[northKey] < h - 0.001f;

// Override: no skirt if neighbor above
if (northAbove)
    northNeedsSkirt = false;

4 Tile Surface (Center-Fan)

Each 1x1m tile gets 5 vertices (4 corners + center) and 4 triangles in a center-fan layout. This ensures clean triangulation without diagonal bias.

topLeft ---- topRight
|  \   N   /  |
|   W  *  E   |
|  /   S   \  |
bottomLeft - bottomRight

// 4 triangles, all sharing center

5 Skirt Generation

For each edge needing a skirt, a quad extends outward by skirtDepth. Skirts are clipped at inner corners (where an adjacent cardinal has no skirt and the height-aware diagonal exists) to prevent overlap.

// North skirt, clipped at corners
nA = topLeft; nB = topRight;
if (!westNeedsSkirt && hasNW_level)
    nA.X += skirtDepth; // clip left
if (!eastNeedsSkirt && hasNE_level)
    nB.X -= skirtDepth; // clip right
if (nB.X - nA.X > 0.001f)
    AddSkirtQuad(nA, nB, northDir);

6 Corner Patches

Outer corners: where two adjacent skirts meet (both cardinals need skirts). Inner corners: "elbow" tiles where both cardinals don't need skirts but the diagonal is absent or lower. Inner corners use randomized tile UVs.

// Outer corner
if (northNeedsSkirt && eastNeedsSkirt)
    AddSkirtCorner(topRight, N, E);

// Inner corner (elbow)
if (!northNeedsSkirt && !eastNeedsSkirt
    && !hasNE_level)
    AddInnerCorner(topRight, N, E);

UV Atlas Layout

All geometry maps into a single texture atlas divided by UV_SPLIT = 0.875:

UV_SPLIT = 0.875 | UV_SKIRT = 0.125
The skirt strip is only 12.5% of texture height, keeping the tile region large for detail.
  • Tile surface (blue): (0, 0.125) to (0.875, 1.0) -- the large square region
  • Skirt strip (yellow): (0, 0) to (0.875, 0.125) -- thin strip at top. The purple/outer edge maps to V=0
  • Corner patch (red): (0.875, 0) to (1.0, 0.125) -- small square at top-right
  • Inner corners reuse a random sub-region of the tile UVs for seamless blending

Height-Aware Diagonal Fix

A critical subtlety: diagonal neighbor checks for skirt clipping and inner corners must account for height. A diagonal tile that exists but is lower doesn't actually cover the gap at the current tile's height.

// Existence-only check (used for other logic)
bool hasNE = tileSet.Contains(neKey);

// Height-aware check (used for clipping + inner corners)
bool hasNE_level = hasNE && heightMap[neKey] >= h - 0.001f;

Without this, a lower diagonal causes:

  • Skirts get clipped (gap appears) but no inner corner fills the hole
  • Visible seams at height transitions in L-shaped or stepped terrain

Method Reference

Generate()

public static ArrayMesh Generate(
    List<Vector3> tilePositions,
    float extensionAmount = 0.15f,
    float skirtDepth = 0.5f,
    float heightOffset = 0.01f,
    float offsetX = -0.5f,
    float offsetZ = -0.5f
)
  • tilePositions: List of (gridX, height, gridZ) for each tile
  • extensionAmount: How far exterior edges extend (unused in current skirt logic, reserved)
  • skirtDepth: Horizontal width of skirt strips
  • heightOffset: Small Y offset to prevent Z-fighting with terrain below
  • offsetX/offsetZ: Shifts tile origin (default -0.5 centers tiles on grid points)

AddSkirtQuad()

Generates a 4-vertex quad extending outward from an edge. Vertices 0,1 are inner (tile edge), 2,3 are outer. UV maps the skirt strip with the purple/outer edge at V=0.

AddSkirtCorner()

Generates a 4-vertex patch filling the gap where two adjacent skirts meet at an outer corner. Uses the corner UV region, rotated 90 degrees CCW.

AddInnerCorner()

Generates a 4-vertex patch at inner "elbow" corners (both cardinals present, diagonal absent/lower). Uses a deterministic random UV position within the tile region, seeded by grid coordinates.
