Resume de la session

Cette session a entierement refactorise le systeme d'initiative de combat. L'ancien systeme utilisait une barre horizontale avec alternance [?][E1][?][E2] et un curseur glissant — mais il buggait quand un pawn mourait (decalage du curseur).

Le nouveau systeme utilise un layout vertical a 2 colonnes, comme une file d'attente tournante :

  • Colonne gauche : un slot vert generique "Joueur" (premiere ligne uniquement)
  • Colonne droite : file d'attente des ennemis vivants
  • Alternance du highlight : seul le slot du joueur actif (joueur OU ennemi) est mis en surbrillance
  • Animation de rotation : quand un ennemi finit son tour, son slot monte + fade out, puis reapparait en bas avec fade in

Commits couverts : c374534 → 0fc055b (creation initiale → refacto complete)

Concept C# : partial class et heritage Control

partial class permet de repartir une classe sur plusieurs fichiers. Godot l'utilise pour generer automatiquement du code « cache » (les exports, les signaux...) dans un fichier separe. Control est la classe de base Godot pour tous les elements d'interface utilisateur (UI). C'est comme dire : « Cette classe est un element visuel de l'ecran ».

Godot : VBoxContainer vs HBoxContainer

L'ancien code utilisait un HBoxContainer (disposition horizontale, comme des cartes posees en ligne sur une table). Le nouveau code utilise un VBoxContainer (disposition verticale, comme une pile de cartes). Chaque ligne de cette pile contient a son tour un HBoxContainer pour placer le slot joueur a gauche et le slot ennemi a droite.

Logique : La file d'attente circulaire

Imaginez une file d'attente au supermarche : quand le premier client est servi, il quitte la tete de la file. Ici c'est pareil mais circulaire — l'ennemi qui vient de jouer ne « part » pas, il retourne a la fin de la file. enemyQueue[0] est toujours le prochain ennemi a jouer. Les flags isFirstPlayerTurn, isPlayerTurnActive et isAnimating controlent l'etat de la machine.

Concept C# : const

const declare une valeur qui ne changera jamais, fixee a la compilation. C'est comme graver un nombre dans la pierre. Ici, la duree de l'animation (0.3 secondes) est constante. Avantage : si on veut ajuster la vitesse, on n'a qu'un seul endroit a modifier.

Godot : GetNodeOrNull et QueueFree — nettoyage securise

GetNodeOrNull cherche un noeud enfant par son nom mais retourne null au lieu de crasher s'il n'existe pas. L'operateur ?. (null-conditional) appelle QueueFree() seulement si le noeud existe. QueueFree() ne detruit pas le noeud immediatement — il le marque pour destruction a la fin de la frame actuelle, evitant les crashs si d'autres references existent encore.

Godot : Construction dynamique d'un layout

Plutot que de definir le layout dans le fichier .tscn (l'editeur visuel), on cree le VBoxContainer entierement en code. C'est comme construire une etagere a la main plutot que d'acheter un meuble en kit. Les proprietes d'ancrage (AnchorLeft/Right = 1.0) le collent au bord droit de l'ecran. GrowHorizontal = Begin fait que le container s'etend vers la gauche quand son contenu grandit.

Pattern : Rebuild complet vs mise a jour partielle

BuildUI() detruit tous les enfants puis reconstruit tout de zero. C'est le pattern le plus simple et robuste : pas besoin de tracker quel slot a change, on repart de zero. Le cout en performance est negligeable pour quelques slots. C'est comme effacer un tableau blanc et reecrire plutot que de gommer mot par mot.

Logique : Construction des lignes

La boucle parcourt chaque ennemi de la file. Pour chaque ennemi, elle cree une ligne horizontale (HBoxContainer). La premiere ligne est speciale (index 0) : elle contient le slot joueur + le slot ennemi. Les lignes suivantes ont un spacer invisible de 50px a gauche pour aligner visuellement la colonne des ennemis.

Logique : Alternance du highlight

La ligne 0 contient le « face a face » : joueur vs ennemi. Le highlight alterne selon isPlayerTurnActive :

• Tour du joueur : playerSlot.SetHighlighted(true), enemySlot.SetHighlighted(false)
• Tour de l'ennemi : playerSlot.SetHighlighted(false), enemySlot.SetHighlighted(true)

Le slot non-highlight est automatiquement attenue (60% d'opacite) grace au code dans TimelineSlotUI.

Godot : Spacer avec CustomMinimumSize

Un Control avec CustomMinimumSize de 50px de large agit comme un spacer invisible. Il « pousse » le slot ennemi vers la droite pour l'aligner avec les ennemis de la ligne 0, creant l'effet de 2 colonnes. C'est l'equivalent d'une cellule vide dans un tableau HTML.

Logique : Machine a etats des tours

OnTurnChanged est appele par BattleInstance a chaque changement de tour. C'est le point d'entree principal. Les etats non-combat (Positioning, EndOfBattle, None) cachent la timeline. Les etats de combat (PlayerTurn, EnemyTurn) l'affichent avec le bon highlight.

Logique : Premier tour vs tours suivants

Le premier PlayerTurn est special : il construit la queue initiale en recuperant tous les ennemis vivants via LINQ (Where + ToList). Les tours suivants declenchent l'animation de rotation. L'astuce : le signal PlayerTurn arrive apres le tour ennemi, donc c'est le bon moment pour faire la rotation (l'ennemi qui vient de jouer passe en bas).

Concept C# : async void — le piege classique

async void permet d'utiliser await dans une methode qui ne retourne rien. C'est comme lancer une tache en arriere-plan sans moyen de savoir quand elle finit (sauf via le flag isAnimating). Normalement on prefere async Task, mais ici c'est un callback d'evenement, donc void est acceptable. Le guard isAnimating empeche de lancer 2 animations en parallele.

Godot : GetChild<Control> — cast type

GetChild(index) retourne un Node generique. Or, pour acceder a Position ou Modulate, il faut un Control (ou un type derive). Le generic GetChild<Control>(1) fait le cast automatiquement. L'index 1 = le 2e enfant de la ligne = le slot ennemi (le 1er etant le slot joueur ou le spacer).

Godot : Tween — animations procedurales

Un Tween interpole une propriete d'un objet entre 2 valeurs sur une duree donnee. SetParallel(true) fait que les 2 tweens (position Y et opacite) s'executent en meme temps plutot que l'un apres l'autre. L'easing Cubic In accelere progressivement — le slot part doucement puis accelere vers le haut, comme un objet qui tombe a l'envers.

Concept C# + Godot : await ToSignal

await ToSignal(tween, "finished") est le pont entre le systeme de signaux Godot et le async/await de C#. C'est comme dire : « Mets ce code en pause et reviens quand l'animation est terminee ». Pendant l'attente, le reste du jeu continue normalement (pas de freeze). C'est la raison d'etre du async sur la methode.

Logique : Fade in du nouveau slot en bas

Apres le rebuild, l'ennemi qui vient de jouer se retrouve en derniere position de enemyQueue, donc en derniere ligne du VBoxContainer. On le rend invisible (Modulate alpha = 0) puis on anime son apparition avec un Cubic Out (commence vite, ralentit a la fin). L'effet global : le slot sort par le haut, reapparait par le bas.

Pattern : Rotation circulaire de liste

PerformRotation prend le premier element, le retire du debut et l'ajoute a la fin. C'est l'implementation la plus simple d'une file circulaire (circular buffer). Complexite : O(n) a cause du RemoveAt(0) qui decale tous les elements. Pour 3-5 ennemis, c'est parfaitement acceptable. On pourrait optimiser avec une LinkedList si necessaire, mais ce serait du sur-engineering.

Logique : Gestion de la mort d'un pawn

Quand un pawn meurt, BattleInstance emet OnPawnDied. Ce handler ne s'interesse qu'aux ennemis (Ownership.ENEMY) — la mort d'un joueur n'affecte pas la timeline. L'ennemi est retire de la queue et l'UI est reconstruite, ce qui fait « remonter » les ennemis restants pour combler le trou. Simple et sans bug de decalage !

Concepts cles de ce fichier

File d'attente circulaire (Circular Queue)

Analogie : C'est comme un manege — quand un ennemi finit de tourner, il retourne a la fin de la file pour attendre son prochain tour. La liste enemyQueue tourne en boucle grace a PerformRotation().

async/await avec Godot Signals

Analogie : C'est comme envoyer un SMS et attendre la reponse — votre vie continue en attendant. await ToSignal() met la methode en pause sans bloquer le jeu, puis reprend quand le signal est emis.

Rebuild pattern (Immediate Mode UI)

Analogie : Plutot que de deplacer les meubles dans une piece, on vide la piece et on la remeuble. BuildUI() detruit tout et reconstruit — simple, robuste, et sans bug d'etat desynchronise.

Guard clauses (isAnimating)

Analogie : Comme un videur a l'entree d'une boite — si une animation est deja en cours, la nouvelle demande est refusee et on fait un fallback sans animation. Empeche les bugs de double animation.

Specificites Godot

QueueFree()

Detruit un noeud a la fin de la frame, pas immediatement. Evite les crashs si d'autres scripts referent encore a ce noeud pendant cette frame.

CreateTween()

Cree un objet Tween attache au noeud appelant. Si le noeud est detruit, le tween est automatiquement annule. SetParallel(true) permet d'enchainer plusieurs proprietes animees simultanement.

PackedScene.Instantiate<T>()

Charge une scene pre-fabriquee (.tscn) et la materialise comme un noeud reel. Le generic <T> caste directement vers le type de script souhaite.

GetNodeOrNull<T>()

Version securisee de GetNode() qui retourne null au lieu de crasher. Indispensable quand on n'est pas sur qu'un noeud existe (par ex. apres migration de scene).

Godot : PanelContainer

PanelContainer est un conteneur qui dessine un fond (un panel) derriere ses enfants. C'est comme un cadre photo — il fournit un fond stylise et contient le portrait + le nom. Le style du fond est defini dans la scene .tscn via un StyleBoxFlat sombre avec coins arrondis.

Godot : _Ready() — l'initialisation Godot

_Ready() est appele automatiquement par Godot quand le noeud et tous ses enfants sont prets dans l'arbre de scene. C'est LE moment pour chercher les noeuds enfants avec GetNode. Si on le faisait dans le constructeur, les enfants n'existeraient pas encore. C'est comme attendre que tous les invites soient arrives avant de commencer la reunion.

Logique : Setup — joueur vert vs ennemi rouge

Avant la refacto, le joueur etait affiche en gris avec « ? » — anonyme et discret. Maintenant il est vert vif avec le texte « Joueur ». Les ennemis restent en rouge avec leur vrai nom. L'operateur ?? (null-coalescing) affiche « ??? » si le pawn n'a pas de nom, ce qui ne devrait jamais arriver mais protege contre les crashs.

Logique : SetHighlighted — le remplacant de SetState

L'ancien code avait 3 etats (Passed, Active, Future) via un enum. Le nouveau code simplifie tout en un booleen : highlight ou pas. Le slot highlight a une opacite de 100%, le slot non-highlight est attenue a 60%. C'est plus simple et plus lisible — on n'a plus besoin de gerer les transitions entre 3 etats.

Godot : StyleBoxFlat dynamique

Un StyleBoxFlat definit l'apparence d'un panneau (couleur, bordure, coins arrondis). Ici on en cree un programmatiquement plutot que dans l'editeur. DrawCenter = false signifie « ne dessine que la bordure, pas le fond » — on veut juste un contour dore de 3px autour du slot actif. AddThemeStyleboxOverride ecrase le style de la scene pour ce noeud uniquement.

Concepts cles de ce fichier

Null-coalescing operator (??)

Analogie : « Si le plat principal est vide, sers le plat de secours ». pawn?.Name ?? "???" retourne le nom du pawn s'il existe, sinon la chaine "???".

Modulate — teinte globale d'un noeud

Analogie : C'est comme mettre un filtre Instagram sur un element. Modulate multiplie toutes les couleurs du noeud et ses enfants. Alpha a 0.6 = 60% d'opacite, rendant le slot translucide quand il n'est pas actif.

Simplification d'enum vers bool

L'ancien TimelineSlotState (3 valeurs) a ete remplace par un simple bool highlighted. Quand 2 des 3 etats produisent le meme resultat visuel, un booleen suffit. Moins de code = moins de bugs.

Specificites Godot

PanelContainer

Conteneur qui fournit un fond dessine derriere ses enfants. Le style du fond est controle par le theme ou par un override via AddThemeStyleboxOverride.

_Ready()

Callback appele une seule fois quand le noeud entre dans l'arbre de scene et que tous ses enfants sont prets. C'est l'equivalent d'un constructeur post-initialisation.

AddThemeStyleboxOverride()

Ecrase le style d'un panneau pour cette instance uniquement, sans modifier le theme global. Les autres Panel du jeu ne sont pas affectes.

Glossaire

File d'attente circulaire (Circular Queue)
Structure de donnees ou le dernier element est suivi du premier, formant une boucle. Ici, les ennemis tournent dans la file apres avoir joue leur tour.
Tween
Objet Godot qui interpole (« tweene ») une propriete entre deux valeurs sur une duree donnee. Utilise pour les animations procedurales (mouvement, opacite, echelle...).
async / await
Mecanisme C# pour ecrire du code asynchrone (qui attend un resultat futur) sans bloquer le thread principal. L'execution est suspendue au await et reprend quand la tache est terminee.
QueueFree()
Methode Godot qui planifie la destruction d'un noeud a la fin de la frame actuelle. Plus sur que la destruction immediate car d'autres scripts peuvent encore referencer le noeud.
PackedScene
Fichier .tscn charge en memoire, pret a etre instancie (materialise) en noeud reel. C'est le « moule » a partir duquel on cree des copies.
Signal
Systeme d'evenements Godot : un noeud emet un signal, d'autres noeuds le recoivent et reagissent. Equivalent du pattern Observer. Ici : OnTurnChanged et OnPawnDied.
StyleBoxFlat
Ressource Godot definissant l'apparence d'un panneau : couleur de fond, bordure, coins arrondis. Peut etre creee en code ou dans l'editeur.
Modulate
Propriete Color de tout CanvasItem Godot. Multiplie les couleurs du noeud et de ses enfants. Utile pour teinter, assombrir ou rendre transparent un element.
LINQ (Where, ToList)
Language Integrated Query — syntaxe C# pour filtrer et transformer des collections. Where(p => ...) filtre, ToList() materialise le resultat en liste concrete.
Null-conditional (?.) et Null-coalescing (??)
?. : n'appelle la methode que si l'objet n'est pas null. ?? : fournit une valeur par defaut si le resultat est null. Deux operateurs pour ecrire du code defensif en une ligne.
partial class
Mot-cle C# permettant de definir une classe sur plusieurs fichiers. Godot l'utilise pour generer automatiquement du code (exports, signaux) dans un fichier cache.
Guard clause
Pattern de code ou les conditions d'erreur/cas speciaux sont verifiees en debut de methode avec un return precoce. Reduit l'imbrication et clarifie le flux principal.
Easing (Cubic In/Out)
Courbes d'acceleration pour les animations. In = demarre lentement et accelere. Out = demarre vite et decelere. Cubic = courbe douce et naturelle.
